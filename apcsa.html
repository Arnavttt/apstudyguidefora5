Integer Division & Modulo most common trap on exam

JAVA int a = 7, b = 2;
int div = a / b; // = 3 (truncates, does NOT round!)
int mod = a % b; // = 1 (remainder)
double real = a / b; // = 3.0 ← STILL INTEGER DIVISION!
double real2 = (double) a / b; // = 3.5 ← cast FIRST operand
double real3 = a / 2.0; // = 3.5 ← literal double forces it

// Modulo use cases:
x % 2 == 0 // even number check
x % 10 // last digit of x
x / 10 // remove last digit of x

Casting widening vs narrowing

JAVA // Widening (automatic — safe, no data loss)
int i = 5;
double d = i; // d = 5.0, automatic

// Narrowing (manual cast required — may lose data)
double x = 9.99;
int n = (int) x; // n = 9 (truncates, does NOT round)

// Common pattern: rounding to nearest int
int rounded = (int) (x + 0.5); // = 10

String Concatenation with Primitives

JAVA int x = 3, y = 4;
System.out.println("Sum: " + x + y); // "Sum: 34" ← LEFT to RIGHT!
System.out.println("Sum: " + (x + y)); // "Sum: 7" ← parens force math first
System.out.println(x + y + " is the sum"); // "7 is the sum" ← + before string
⚡ Exam Trap

When + is used left-to-right and hits a String, everything after concatenates. When it hits two ints first, it adds them. Know the order of operations.

String Methods — Complete Reference

  • s.length()Returns int — number of characters. Note: method call, NOT .length like arrays!
  • s.substring(i, j)Returns String from index i (inclusive) to j (exclusive). s.substring(2,5) of "abcdef" → "cde"
  • s.substring(i)Returns String from index i to end. s.substring(3) of "abcdef" → "def"
  • s.indexOf(str)Returns first index where str occurs, or -1 if not found.
  • s.equals(other)Returns boolean. Always use this, NEVER == for String content comparison.
  • s.compareTo(other)Returns negative (s before other), 0 (equal), positive (s after other) — lexicographic.
  • s.toUpperCase()Returns new String in uppercase. Strings are immutable — original unchanged.
  • s.toLowerCase()Returns new String in lowercase.
STRING EXAMPLES String s = "Hello, World";
s.length() // 12
s.substring(0, 5) // "Hello"
s.substring(7) // "World"
s.indexOf("o") // 4 (first occurrence)
s.indexOf("z") // -1 (not found)
s.equals("Hello, World") // true

// Getting last character:
char last = s.charAt(s.length() - 1); // charAt is NOT on AP reference but appears in code

// Loop through each character:
for (int i = 0; i < s.length(); i++) {
  String ch = s.substring(i, i+1); // substring preferred on AP exam
}

De Morgan's Laws tested directly on exam

DE MORGAN'S // !(A && B) is equivalent to (!A || !B)
// !(A || B) is equivalent to (!A && !B)

// Example:
!( x > 0 && y > 0 ) // same as:

!( x == 5 || x == 10 ) // same as:
( x != 5 && x != 10 ) // flip OR to AND, flip == to !=

Writing Classes
// Unit 5 · ~5–7.5% of exam
FRQ Type: Writing a class from scratch

Anatomy of a Java Class

JAVA public class BankAccount {

  // 1. Instance variables (private — always!)
  private String owner;
  private double balance;

  // 2. Constructor (same name as class, no return type)
  public BankAccount(String owner, double initialBalance) {
    this.owner = owner; // this.field = parameter
    this.balance = initialBalance;
  }

  // 3. Accessor (getter) — returns value, does NOT modify
  public double getBalance() {
    return balance;
  }

  // 4. Mutator (setter) — modifies state, usually void
  public void deposit(double amount) {
    if (amount > 0) balance += amount;
  }

  // 5. toString — always override this for FRQs
  public String toString() {
    return owner + ": $" + balance;
  }
}

Static vs. Instance critical distinction

Instance (non-static)

  • Belongs to a specific object
  • Called on an object: obj.method()
  • Can access instance variables
  • Different objects have different values

Static

  • Belongs to the class, not an object
  • Called on class: ClassName.method()
  • Cannot access non-static instance variables
  • Shared across all instances: Math.sqrt()
⚡ Null Reference Trap

Declaring an object variable does NOT create the object. BankAccount b; → b is null. Calling b.deposit(10) throws NullPointerException. Must use new: BankAccount b = new BankAccount("Alice", 0);

Array Traversal Patterns

JAVA int[] arr = {3, 7, 1, 9, 4};

// FORWARD traversal (standard)
for (int i = 0; i < arr.length; i++) {
  System.out.println(arr[i]);
}

// BACKWARD traversal
for (int i = arr.length - 1; i >= 0; i--) {
  System.out.println(arr[i]);
}

// FOR-EACH (no index — read only)
for (int val : arr) {
  System.out.println(val); // can't use val to modify arr
}

// PARTIAL traversal (first half)
for (int i = 0; i < arr.length / 2; i++) { ... }

Array Algorithms — FRQ Staples

JAVA — SHIFTING ELEMENTS // Shift all elements LEFT (remove first)
for (int i = 0; i < arr.length - 1; i++) {
  arr[i] = arr[i + 1];
}
arr[arr.length - 1] = 0; // clear last slot

// Shift elements RIGHT to insert at index k
for (int i = arr.length - 1; i > k; i--) { // go BACKWARD!
  arr[i] = arr[i - 1];
}
arr[k] = newValue;
⚡ Arrays.length is a field — no ()
  • arr.length ← correct (field)
  • arr.length() ← compile error
  • str.length() ← correct for String (method)
  • This distinction is tested every single year

Removing During Iteration — Critical Pattern

❌ Wrong — Skips elements after removal
for (int i = 0; i < list.size(); i++) {
  if (list.get(i) < 0) list.remove(i); // i++ skips next element after removal!
}
✅ Correct — Iterate backward OR don't increment when removing
// Option 1: backward iteration
for (int i = list.size() - 1; i >= 0; i--) {
  if (list.get(i) < 0) list.remove(i);
}

// Option 2: don't increment i when removing
int i = 0;
while (i < list.size()) {
  if (list.get(i) < 0) list.remove(i);
  else i++; // only advance if NOT removed
}

2D Array Traversal Patterns

JAVA — ALL TRAVERSAL PATTERNS // ROW-MAJOR order (row by row) — most common
for (int r = 0; r < grid.length; r++) {
  for (int c = 0; c < grid[r].length; c++) {
    System.out.print(grid[r][c] + " ");
  }
  System.out.println();
}

// COLUMN-MAJOR order (column by column)
for (int c = 0; c < grid[0].length; c++) {
  for (int r = 0; r < grid.length; r++) {
    System.out.print(grid[r][c] + " ");
  }
}

// ENHANCED for-each (read-only)
for (int[] row : grid) {
  for (int val : row) {
    System.out.print(val + " ");
  }
}

// SUM all elements
int total = 0;
for (int[] row : grid)
  for (int val : row)
    total += val;

// SUM a specific COLUMN c
int colSum = 0;
for (int r = 0; r < grid.length; r++)
  colSum += grid[r][c];

// SUM a specific ROW r
int rowSum = 0;
for (int c = 0; c < grid[r].length; c++)
  rowSum += grid[r][c];
💡 Dimensions Memory Trick

grid.length = number of ROWS (how many arrays inside the outer array)
grid[0].length = number of COLUMNS (length of the first inner array)
Think of it as: rows = how many, cols = how wide

Inheritance
// Unit 9 · ~5–10% of exam
FRQ Type: Extend a class / implement interface

extends, super, @Override

JAVA public class Animal {
  private String name;

  public Animal(String name) { this.name = name; }

  public String getName() { return name; }

  public String sound() { return "..."; }

  public String toString() { return name + " says " + sound(); }
}

public class Dog extends Animal { // Dog IS-A Animal
  private String breed;

  public Dog(String name, String breed) {
    super(name); // MUST be first line — calls Animal constructor
    this.breed = breed;
  }

  @Override
  public String sound() { return "Woof"; } // overrides Animal.sound()

  public String getBreed() { return breed; }
}

Polymorphism & Dynamic Dispatch

JAVA — POLYMORPHISM Animal a = new Dog("Rex", "Lab"); // Animal reference, Dog object

a.sound() // "Woof" — Dog's version (runtime type determines method)
a.getName() // "Rex" — Animal method, works fine
a.getBreed() // COMPILE ERROR — Animal reference doesn't know about breed

// Cast to access subclass-specific methods
((Dog) a).getBreed() // "Lab" — downcast works if a is actually a Dog

// instanceof check before casting (safe pattern)
if (a instanceof Dog) {
  Dog d = (Dog) a;
  d.getBreed();
}

Abstract Class

Recursion
// Unit 10 · ~5–7.5% of exam
Trace recursive calls — always on exam

Recursion Structure — Every Recursive Method Has These 2 Parts

JAVA public static int factorial(int n) {
  // BASE CASE — stops recursion (MUST have at least one)
  if (n == 0 || n == 1) return 1;

  // RECURSIVE CASE — calls itself with SMALLER input
  return n * factorial(n - 1); // n-1 moves TOWARD base case
}

// Trace: factorial(4)
// = 4 * factorial(3)
// = 4 * 3 * factorial(2)
// = 4 * 3 * 2 * factorial(1)
// = 4 * 3 * 2 * 1 ← base case hit
// = 24

Classic Recursive Methods

JAVA — FIBONACCI + BINARY SEARCH + REVERSE STRING // Fibonacci
public static int fib(int n) {
  if (n <= 1) return n; // base cases: fib(0)=0, fib(1)=1
  return fib(n-1) + fib(n-2); // TWO recursive calls
}

// Reverse a string recursively
public static String reverse(String s) {
  if (s.length() <= 1) return s;
  return reverse(s.substring(1)) + s.substring(0, 1);
}

// Sum of array recursively
public static int sum(int[] arr, int index) {
  if (index == arr.length) return 0; // base case
  return arr[index] + sum(arr, index + 1);
}
⚡ Stack Overflow = Missing/Unreachable Base Case

If a recursive method never reaches its base case, you get infinite recursion → StackOverflowError. On the exam, trace calls and count how many times before the base case is hit.

What is the value of x after executing: int x = 17 / 5 * 2 + 17 % 5;

Answer: C) 8

Given String s = "programming";, what does s.substring(3, 7) return?

  • A) "prog"
  • B) "gram"
  • C) "gramm"
  • D) "ramm"
Answer: B) "gram"
"programming" — indices: p(0) r(1) o(2) g(3) r(4) a(5) m(6) m(7) i(8) n(9) g(10)
substring(3,7) = chars at 3,4,5,6 = "gram". End index 7 is EXCLUSIVE.
Q3Boolean Logic / De Morgan's · Unit 3

Which expression is equivalent to !(a > 5 || b == 3)?

  • A) a < 5 || b != 3
  • B) a <= 5 && b != 3
  • C) a <= 5 || b != 3
  • D) a < 5 && b != 3
Answer: B) De Morgan's Law: !(P || Q) ≡ !P && !Q
!(a > 5) = (a <= 5) and !(b == 3) = (b != 3), connected by &&
Q4Loop Tracing · Unit 4

How many times does the body of this loop execute?

for (int i = 1; i < 20; i *= 2) { ... }
  • A) 4
  • B) 5
  • C) 19
  • D) 20
Answer: B) 5
i values: 1, 2, 4, 8, 16 (all < 20, execute body) → next i = 32 ≥ 20, loop exits. Count = 5.

  • B) 9
  • C) 10
  • D) 13
Answer: C) 10
m[2][0] = row 2, col 0 = 7. m[0][2] = row 0, col 2 = 3. Sum = 10.
Q7Inheritance / Polymorphism · Unit 9

Assume Cat extends Animal and overrides speak(). Given Animal a = new Cat();, what does a.speak() call?

  • A) Animal's speak()
  • B) Cat's speak()
  • C) Compile error
  • D) Both Animal's and Cat's speak()
Answer: B) Cat's speak()
Dynamic dispatch: Java determines which overridden method to call at runtime based on the actual object type (Cat), not the reference type (Animal). This is the core of polymorphism.
Q8Recursion · Unit 10

What does mystery(4) return?

public static int mystery(int n) {
  if (n <= 0) return 0;
  return n + mystery(n - 2);
}
  • A) 4
  • B) 5
  • C) 6
  • D) 10
Answer: C) 6
mystery(4) = 4 + mystery(2) = 4 + 2 + mystery(0) = 4 + 2 + 0 = 6
(decrements by 2 each call; hits base case when n=0)
Q9Casting · Unit 1

What is printed?

double x = 7.9;
int y = (int) x;
System.out.println(y);
  • A) 8
  • B) 7
  • C) 7.9
  • D) Compile error
Answer: B) 7
Casting to int always truncates (cuts off decimal), it does NOT round. 7.9 → 7. To round, use (int)(x + 0.5).
Q10String Immutability · Unit 2

What is the value of s after these statements?

String s = "hello";
s.toUpperCase();
System.out.println(s);
  • A) "hello"
  • B) "HELLO"

FRQ Type 1 Methods & Control Structures — WordAnalyzer

A class WordAnalyzer is used to analyze strings. You will write two methods of this class. It has a single String instance variable word, initialized in its constructor.

Part A — 3 pts

Write the method countVowels() that returns the number of vowels (a, e, i, o, u — upper or lowercase) in word.

Part B — 4 pts

Write the method hasConsecutiveDuplicates() that returns true if the word contains two identical consecutive characters (e.g., "hello" → true because "ll"), and false otherwise.

SOLUTION — PART A public int countVowels() {
  int count = 0;
  String vowels = "aeiouAEIOU";
  for (int i = 0; i < word.length(); i++) {
    String ch = word.substring(i, i + 1);
    if (vowels.indexOf(ch) != -1) count++; // -1 means not found
  }
  return count;
}
SOLUTION — PART B public boolean hasConsecutiveDuplicates() {
  for (int i = 0; i < word.length() - 1; i++) { // length-1: compare i and i+1
    String curr = word.substring(i, i + 1);
    String next = word.substring(i + 1, i + 2);
    if (curr.equals(next)) return true;
  }
  return false;
}
Grading Notes
  • Part A: +1 loop structure, +1 correct vowel check, +1 count returned
  • Part B: +1 loop to length-1, +1 correct character access, +1 correct comparison (.equals), +1 return false at end
  • Common mistake: using == to compare strings — 1 pt deduction
FRQ Type 2 Class — Write a complete class: ShoppingCart

Write a complete class ShoppingCart that models an online shopping cart.

Specification

• Instance variables: ArrayList<String> items and double totalPrice
• Constructor: no-arg, initializes empty cart
• addItem(String name, double price): adds item name to list, adds price to total
• removeItem(String name, double price): removes first occurrence of name; subtracts price if found; returns true if removed, false if not found
• getTotal(): returns totalPrice
• toString(): returns items list + total, e.g., "[Milk, Bread] Total: $5.49"

COMPLETE SOLUTION import java.util.ArrayList;

public class ShoppingCart {
  private ArrayList<String> items;
  private double totalPrice;

  public ShoppingCart() {
    items = new ArrayList<>();
    totalPrice = 0.0;
  }

  public void addItem(String name, double price) {
    items.add(name);
    totalPrice += price;
  }

  public boolean removeItem(String name, double price) {
    for (int i = 0; i < items.size(); i++) {
      if (items.get(i).equals(name)) {
        items.remove(i);
        totalPrice -= price;
        return true;
      }
    }
    return false;
  }

  public double getTotal() { return totalPrice; }

  public String toString() {
    return items.toString() + " Total: $" + totalPrice;
  }
}
FRQ Type 3 Array / ArrayList — Grade Processor

The following method is part of a GradeBook class. An int[] called grades stores exam scores.

Part A — 4 pts

Write getAboveAverage(int[] grades) that returns a new ArrayList<Integer> containing only scores that are strictly above the average of all scores.

Part B — 3 pts

Write removeFailingScores(ArrayList<Integer> scores) that removes all scores below 60 from the list. Modifies the list in place, returns nothing.

PART A SOLUTION public static ArrayList<Integer> getAboveAverage(int[] grades) {
  // Step 1: compute average
  int sum = 0;
  for (int g : grades) sum += g;
  double avg = (double) sum / grades.length; // cast for decimal average!

  // Step 2: collect above-average
  ArrayList<Integer> result = new ArrayList<>();
  for (int g : grades) {
    if (g > avg) result.add(g);
  }
  return result;
}
PART B SOLUTION — BACKWARD ITERATION TO AVOID SKIP BUG public static void removeFailingScores(ArrayList<Integer> scores) {
  for (int i = scores.size() - 1; i >= 0; i--) {
    if (scores.get(i) < 60) {
      scores.remove(i); // remove by index — safe when going backward
    }
  }
}
FRQ Type 4 2D Array — Image Processor

A grayscale image is represented as a 2D array of ints where each value (0–255) represents pixel brightness. Higher = brighter.

Part A — 3 pts

Write brightenRow(int[][] image, int row, int amount) that increases every pixel in the given row by amount, but caps values at 255. Modifies in place.

Part B — 4 pts

Write countDarkPixels(int[][] image, int threshold) that returns the number of pixels in the entire image with value strictly less than threshold.

PART A SOLUTION public static void brightenRow(int[][] image, int row, int amount) {
  for (int c = 0; c < image[row].length; c++) {
    image[row][c] += amount;
    if (image[row][c] > 255) image[row][c] = 255; // cap at 255
    // Alternative: image[row][c] = Math.min(image[row][c] + amount, 255);
  }
}
PART B SOLUTION public static int countDarkPixels(int[][] image, int threshold) {
  int count = 0;
  for (int r = 0; r < image.length; r++) {
    for (int c = 0; c < image[r].length; c++) {
      if (image[r][c] < threshold) count++;
    }
  }
  return count;
}
FRQ Grading Insight
  • Part A: +1 correct loop, +1 add amount, +1 cap logic correct
  • Part B: +1 outer loop, +1 inner loop, +1 condition, +1 return count
  • Using enhanced for-each is fine for Part B (read-only), but Part A REQUIRES index-based access to modify elements

⚡ Perfect Score Checklist

  • Integer division truncates — never rounds. (int)(7.9) = 7. Know this cold.
  • String equality = .equals() — never == for content. Tested every year.
  • Array: .length (no parens). String: .length() (with parens). ArrayList: .size() (with parens).
  • ArrayList requires wrapper types — Integer, Double, Boolean, NOT int, double, boolean.
  • 2D array: grid[row][col] — rows first, always. grid.length = rows. grid[0].length = cols.
  • For-each cannot modify array elements — val is a copy, changing it doesn't change the array.
  • Remove from ArrayList while iterating: go backward or use while loop without incrementing.
  • super() must be the first line of a subclass constructor — always.
  • Polymorphism: runtime type determines which overridden method runs — not the reference type.
  • Recursion needs a base case that is actually reachable. Trace every recursive call on the exam.
  • Strings are immutable — toUpperCase(), substring(), etc. return NEW strings, original unchanged.
  • null reference — declaring an object variable without new gives null; calling any method on it = NullPointerException.
