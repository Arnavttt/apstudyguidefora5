Big Idea 1 · ~10–13% of exam
Creative Development

Collaboration

Effective collaboration in computing: diverse perspectives reduce bias and improve solutions. Pair programming, code review, version control (Git). Computing innovations are collaborative — acknowledge contributions and limitations.

Program Development

Iterative process: investigate → design → prototype → test → reflect → iterate. Errors: syntax (code won't run), runtime (crashes during execution), logic (runs but wrong result). Debugging = systematic process.

Documentation

Comments explain WHY not just what. Good documentation enables collaboration, maintenance, and reuse. Variables/functions should have meaningful names. Abstraction hides implementation details.

Big Idea 2 · ~17–22% of exam
Data

Binary & Data Representation

BINARY: base-2 number system (0s and 1s)
1 bit = one binary digit. 1 byte = 8 bits.
Binary 1010 = 1×8 + 0×4 + 1×2 + 0×1 = 10 in decimal

DATA TYPES:
Text: ASCII (7-bit, 128 chars) → Unicode (millions of chars)
Images: pixels = color values (RGB triplets). More bits per pixel = higher color depth.
Sound: samples = measurements of wave amplitude over time. Higher sample rate = better quality.
Video: sequence of image frames + audio track.

DATA COMPRESSION:
Lossless: no data lost (ZIP, PNG, FLAC) — exact original recoverable
Lossy: some data permanently removed (JPEG, MP3, MP4) — smaller file, lower quality

Data Analysis

Metadata

Data about data. EXIF data in photos (location, camera settings, timestamp). Enables searching and organization. Privacy concern: metadata can reveal sensitive info even if content seems innocuous.

Bias in Data

Sampling bias: non-representative sample → biased conclusions. Confirmation bias: seeking data that confirms existing beliefs. Algorithmic bias: training data reflects historical inequalities → biased predictions (facial recognition, hiring algorithms).

Visualization

Choosing wrong chart type can mislead. Bar charts for comparison, line graphs for trends over time, scatter plots for correlation. Truncated y-axis exaggerates differences. Statistical significance ≠ practical significance.

Big Idea 3 · ~30–35% of exam
Algorithms & Programming

Pseudocode Reference — AP CSP Style

VARIABLES & ASSIGNMENT:
x ← 5 // assign value to variable
text ← "hello"

CONDITIONS:
IF (condition) { // if-else structure
  statements
} ELSE {
  statements
}

LOOPS:
REPEAT n TIMES { } // repeat n times
REPEAT UNTIL (cond) { } // loop until condition true

LISTS:
myList ← [1, 2, 3, 4]
myList[1] // index starts at 1 in AP CSP!
APPEND(myList, 5) // adds to end
LENGTH(myList) // returns 4

PROCEDURES:
PROCEDURE myProc(param) {
  RETURN value
}

Algorithm Concepts

Sequencing, Selection, Iteration

All algorithms are built from these three. Sequencing = order matters. Selection (if/else) = decision. Iteration (loops) = repetition. Any algorithm can be expressed with just these three.

Linear vs. Binary Search

Linear: check each element — O(n). Works on unsorted lists. Binary: divide in half each time — O(log n). Requires sorted list. Binary search is dramatically faster for large datasets.

Undecidable Problems

Halting problem: impossible to write a program that correctly determines for ALL programs whether they will halt or run forever. Some problems are computationally intractable — no efficient solution exists.

Efficiency & Heuristics

Not all problems can be solved efficiently. Heuristics find good-enough solutions quickly (GPS routing, game AI). Reasonable time = polynomial time. Exponential time algorithms impractical for large inputs.

Big Idea 4 · ~11–15% of exam
Computing Systems & Networks

The Internet — How It Works

IP Addresses & DNS

Every device has IP address (IPv4: 32-bit; IPv6: 128-bit). DNS translates domain names (google.com) to IP addresses. DHCP assigns IP addresses dynamically. Routers direct packets between networks.

Packets & Protocols

Data split into packets. Each packet: header (destination, source, sequence number) + data. TCP: reliable, ordered (web, email). UDP: fast, no guarantee (video streaming, gaming). Fault-tolerant: packets can take different routes.

HTTP & HTTPS

HTTP: transfers web pages. HTTPS: encrypted with TLS/SSL. Browser sends request → server responds with content. Status codes: 200 (OK), 404 (not found), 500 (server error).

Cybersecurity

Phishing: fake communications steal credentials. Malware: viruses, ransomware, spyware. DDoS: flood server with requests. Defense: encryption, multi-factor authentication, software updates, firewalls. No system is perfectly secure.

Big Idea 5 · ~21–26% of exam
Impact of Computing

Privacy & Data

PII (Personally Identifiable Information): name, SSN, location, biometrics. COPPA protects children's data online. Terms of service grant companies data rights. Metadata reveals behavior patterns.

Intellectual Property

Copyright: automatic protection. Creative Commons: flexible licensing. Open source: source code freely available. Plagiarism = using others' work without credit. Fair use: education, commentary, transformative use.

Digital Divide

Unequal access to computing and internet. Factors: economic, geographic, disability, age, education. Exacerbates existing inequality. Solutions: subsidized broadband, digital literacy programs, device access programs.

Beneficial & Harmful Impacts

Same technology can be beneficial and harmful. Surveillance: safety vs. civil liberties. Social media: connection vs. misinformation. AI in medicine: diagnosis vs. bias. Automation: efficiency vs. job displacement.

Performance Task · 30% of exam score
Create Performance Task
Create PT Requirements
  • What you submit: Program file + video (≤1 min showing program running + explaining 1 feature) + Written Responses
  • Row 1 — Program Purpose & Function: Describe what program does, what the video shows, how it works
  • Row 2 — Data Abstraction: Show a list/data structure, explain how it stores data, why a list was used (not individual variables)
  • Row 3 — Managing Complexity: Explain how your list MANAGES complexity — what would code look like without the list? Show it would be harder.
  • Row 4 — Procedural Abstraction: Show a procedure you wrote with a parameter. Explain what it does and how it contributes to program function.
  • Row 5 — Algorithm Implementation: Procedure must include sequencing + selection (if) + iteration (loop). Explain step-by-step.
  • Row 6 — Testing: Describe 2 different test cases (different inputs). What was the condition tested, expected result, and actual result.

⚡ Perfect Score Checklist

  • AP CSP lists are 1-indexed — myList[1] is the first element (unlike most languages)
  • Lossless vs. lossy compression: lossless = exact recovery (ZIP, PNG); lossy = permanent data loss (JPEG, MP3)
  • Binary search requires sorted data; linear search works on anything
  • Halting problem = undecidable — cannot write program to determine if all programs halt
  • Internet is fault-tolerant: packets can route around failures via different paths
  • Create PT: your procedure MUST have a parameter and include all three: sequence + selection + iteration
  • Data abstraction: use a list AND explain why a list manages complexity better than individual variables
  • Privacy: same data can be beneficial (research) and harmful (surveillance)
  • Digital divide = unequal access to computing — not just hardware but skills and connectivity
  • Open source ≠ free: free as in freedom (to use, modify, distribute), may cost money
