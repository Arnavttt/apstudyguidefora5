Computing Innovations and Their Impacts

Learn on this site
Beneficial and Harmful Impacts

Every computing innovation has intended and unintended effects. Social media: connects people (intended) and spreads misinformation (unintended). GPS: enables navigation and enables surveillance. Evaluating both sides is essential.

Common misconception: "Unintended effects" does not mean the creators were irresponsible — complex systems produce emergent effects that are difficult to predict.

How it can be tested: Identify intended and unintended effects of a computing innovation; evaluate trade-offs.

Responsible Innovation

Programmers have responsibility for the impacts of their software. Computing can perpetuate or reduce inequality. Accessibility: designing for users with disabilities is both ethical and improves usability for all.

Common misconception: Accessibility features often benefit everyone: captions help in noisy environments; voice interfaces help drivers; large text helps anyone in sunlight.

How it can be tested: Explain programmer responsibility; describe how accessibility improves software for all users.

Global Impacts

Computing innovations have global effects. Remote work enabled by computing changed labor markets. Automation displaces some jobs while creating others. Computing power is not equally distributed globally.

Common misconception: Automation does not simply eliminate jobs — it changes the nature of work. New technologies historically create new categories of jobs.

How it can be tested: Describe how a computing innovation affects global communities; identify winners and losers.

Key Vocabulary
  • unintended consequence — Effect of a computing innovation that was not planned or anticipated by creators
  • intended effect — The purpose a computing innovation was designed to achieve
  • accessibility — Designing software usable by people with disabilities (visual, motor, auditory)
  • automation — Use of computing to perform tasks previously done by humans
  • digital divide — Gap between those with access to technology and those without
Objectives
  • Identify intended and unintended effects of a computing innovation.
  • Explain programmer responsibility for software impacts.
  • Describe how automation affects employment and society.
How this can be tested
  • Written response: analyze benefits and harms of a specific computing innovation.
  • MCQ: identify an unintended consequence of a described innovation.
  • Written response: explain one positive and one negative social impact of social media.
Practice before moving on
  • Analyze the intended and unintended effects of facial recognition technology.
  • Research an example of accessibility features that help all users.

Intellectual Property and Licensing

Learn on this site
Copyright and Fair Use

Copyright: creator's exclusive legal right to reproduce, distribute, and adapt their work. Fair use: limited use without permission for commentary, education, or parody. Internet created new IP challenges.

Common misconception: Copyright is automatic — you do not need to register or add © to have copyright protection.

How it can be tested: Identify copyrighted works; determine if a use might qualify as fair use; explain the limits of fair use.

Creative Commons and Open Source

Creative Commons (CC) licenses allow creators to grant specific rights. CC-BY: share with credit. CC-NC: non-commercial only. Open source: source code freely available (e.g., Linux, Mozilla). Not the same as public domain.

Common misconception: Open source is not free of restrictions — many open source licenses require attribution or prohibit commercial use. Always check the specific license.

How it can be tested: Identify Creative Commons license restrictions; explain open source vs proprietary software.

Plagiarism and Ethical Use

Plagiarism: presenting someone else's work as your own. Applies to code, text, and ideas. Ethical use of computing includes crediting sources and not misrepresenting work.

Common misconception: Copying code from Stack Overflow or GitHub is not automatically plagiarism, but passing it off as your own in an academic context is.

How it can be tested: Distinguish plagiarism from legitimate collaboration; apply ethical standards to code reuse.

Key Vocabulary
  • copyright — Legal right giving creators control over reproduction and distribution of their work
  • fair use — Doctrine allowing limited use of copyrighted material without permission
  • Creative Commons — Licenses that allow creators to grant specific permissions (share, remix, commercial)
  • open source — Software with publicly available source code; may have license restrictions
  • public domain — Works with no copyright restrictions; anyone can use freely
Objectives
  • Explain copyright and fair use.
  • Identify restrictions of Creative Commons license types.
  • Distinguish open source from public domain.
How this can be tested
  • MCQ: identify whether a use qualifies as fair use.
  • MCQ: identify what a CC-BY-NC license permits.
  • Written response: explain the difference between open source and public domain.
Practice before moving on
  • Find the license of a popular open source project (e.g., Linux) and summarize restrictions.
  • Evaluate whether a described use of an image qualifies as fair use.

Privacy and Surveillance

Learn on this site
Data Privacy Principles

Privacy: control over your own personal information. Computing enables mass data collection. Principles: data minimization (collect only what's needed), purpose limitation (use only for stated purpose), informed consent.

Common misconception: Consent is not truly "informed" when privacy policies are 10,000 words that no user reads. This is an ongoing ethical and regulatory challenge.

How it can be tested: Explain data minimization and purpose limitation; describe the limits of consent as a privacy mechanism.

Surveillance and Tracking

Web tracking: cookies, pixel tracking, device fingerprinting. Government surveillance: collection of phone metadata, content of communications. Facial recognition used in law enforcement — accuracy varies by demographic.

Common misconception: Even metadata (not content) is extremely revealing — pattern of calls, locations, timing reveals relationships and activities.

How it can be tested: Explain web tracking mechanisms; describe privacy risks from surveillance; evaluate facial recognition accuracy concerns.

Legal Protections and Trade-offs

GDPR (EU): right to erasure, data portability, breach notification. COPPA (US): limits data collection on children under 13. Encryption protects privacy but can obstruct lawful investigation. Privacy vs security trade-off is ongoing.

Common misconception: Privacy and security are often framed as opposites but they can reinforce each other — strong encryption protects citizens from criminals as well as from governments.

How it can be tested: Describe GDPR rights; explain the privacy-security trade-off; evaluate arguments on both sides.

Key Vocabulary
  • data minimization — Collecting only the personal data that is necessary for the stated purpose
  • GDPR — EU regulation granting rights to data access, deletion, and portability
  • cookie — Small file stored by browser tracking user activity across sessions or sites
  • facial recognition — AI system matching faces to identities; accuracy varies significantly by demographic
  • COPPA — US law limiting data collection on users under age 13
Objectives
  • Explain data minimization, purpose limitation, and informed consent.
  • Describe web tracking mechanisms and their privacy implications.
  • Explain the privacy-security trade-off with arguments on both sides.
How this can be tested
  • Written response: describe a privacy risk from a specific technology.
  • MCQ: identify a GDPR right.
  • Written response: evaluate the trade-off between encryption and law enforcement access.
Practice before moving on
  • Check what cookies a website sets using browser developer tools.
  • Evaluate the argument for and against end-to-end encryption.

Ethics and the Future of Computing

Learn on this site
Algorithmic Bias

Algorithms can reflect the biases in their training data. Hiring algorithms, criminal risk scores, and loan approvals can perpetuate historical discrimination. Explainability: understanding why an algorithm made a decision.

Common misconception: An algorithm is not "objective" just because it is mathematical — it reflects the data it was trained on, including historical biases.

How it can be tested: Identify how algorithmic bias occurs; explain the importance of explainability in high-stakes AI decisions.

AI and Machine Learning

AI learns patterns from data. Machine learning: computers improve at tasks from experience without explicit programming. Applications: image recognition, language translation, recommendation systems, autonomous vehicles.

Common misconception: AI is not "thinking" like humans — it is pattern matching at massive scale. Current AI lacks general reasoning; it is narrow AI.

How it can be tested: Describe how machine learning works; identify limitations and ethical concerns with AI systems.

Future of Computing and Human Values

Computing evolves rapidly. Quantum computing: potential to break current encryption. Moore's Law slowing. Computing must be guided by human values: fairness, privacy, accessibility, accountability.

Common misconception: Quantum computing breaking encryption is a future threat, not an immediate one. Post-quantum cryptography is being developed to address it.

How it can be tested: Explain quantum computing's potential impact; describe how human values should guide computing development.

Key Vocabulary
  • algorithmic bias — When an AI system produces outcomes that systematically favor or disadvantage certain groups
  • machine learning — AI technique where systems learn patterns from data rather than explicit rules
  • explainability — Ability to understand and explain why an AI made a specific decision
  • narrow AI — AI specialized for one task; cannot generalize to others; all current AI is narrow
  • quantum computing — Computing using quantum mechanical phenomena; potential to break RSA encryption
Objectives
  • Explain algorithmic bias and how it arises from training data.
  • Describe machine learning and two ethical concerns with AI.
  • Explain why human values must guide computing development.
How this can be tested
  • Written response: describe an ethical concern with AI in criminal justice.
  • MCQ: identify algorithmic bias in a described scenario.
  • Written response: explain what "explainability" means and why it matters.
Practice before moving on
  • Research one AI system (e.g., COMPAS, Amazon Rekognition) and its documented bias.
  • Debate: should AI be used in high-stakes decisions (hiring, bail)?
