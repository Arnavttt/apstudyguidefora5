1D Arrays

Learn on this site
Array Declaration and Creation

int[] arr = new int[5]; creates an array of 5 ints (indices 0–4). Default values: 0 for numeric, false for boolean, null for objects. arr.length gives the number of elements.

Common misconception: Array indices go from 0 to arr.length-1. Accessing arr[arr.length] throws ArrayIndexOutOfBoundsException (AIOOBE).

How it can be tested: Declare and create arrays; access elements by index; avoid AIOOBE.

Array Initialization and Access

Initialize inline: int[] scores = {90, 85, 78, 92}; Access: scores[0] = 90. Modify: scores[2] = 100; Arrays have fixed size — cannot grow.

Common misconception: Confusing array.length (field, no parentheses) with String.length() (method, needs parentheses) is a frequent error.

How it can be tested: Initialize arrays with values; read and modify individual elements; use arr.length correctly.

Array Traversal

Traverse with for: for(int i=0; i

Common misconception: Use enhanced for when you only need to read values; use indexed for when you need the index or need to modify elements.

How it can be tested: Write traversals to compute sum, max, min; identify when to use indexed vs enhanced for.

Key Vocabulary
  • array — Fixed-size collection of same-type elements stored contiguously; indexed 0 to length-1
  • arr.length — Field (no parentheses) giving number of elements in the array
  • ArrayIndexOutOfBoundsException — Runtime error when accessing index < 0 or >= arr.length
  • enhanced for — for(type var : array) — iterates all elements; read-only; no index
  • array initializer — {val1, val2, val3} — creates and populates an array inline
Objectives
  • Declare, create, and initialize 1D arrays of primitive and reference types.
  • Traverse arrays with indexed and enhanced for loops.
  • Compute aggregate values (sum, max, count) using an array.
How this can be tested
  • FRQ: write a method that traverses an array to find the maximum.
  • MCQ: trace array traversal to find the final state or output.
  • MCQ: identify the AIOOBE scenario.
Practice before moving on
  • Write a method that returns the sum and the average of an int array.
  • Trace an array traversal that finds the first element greater than a threshold.

ArrayList

Learn on this site
ArrayList vs Array

ArrayList is a resizable list from java.util. Key methods: add(E), remove(int), get(int), set(int,E), size(). Must import java.util.ArrayList; Requires wrapper types: ArrayList not ArrayList.

Common misconception: ArrayList size() is a method (parentheses); array length is a field (no parentheses). Mixing them up is a compile error.

How it can be tested: Choose ArrayList when size varies; choose array when size is fixed or performance critical.

ArrayList Methods

add(E e): appends; add(int i, E e): inserts at i shifting others. remove(int i): removes at i (shifting); get(i): returns element; set(i,E): replaces; size(): count. Iterating while modifying can skip elements.

Common misconception: Removing an element at index i shifts later elements left. If you then increment i, you skip the element that moved into position i. Iterate backwards or use iterator when removing.

How it can be tested: Write loops that add to, remove from, and search an ArrayList.

Traversing ArrayList

Standard for: for(int i=0; i

Common misconception: Do not use an enhanced for loop to add or remove elements — causes ConcurrentModificationException.

How it can be tested: Write ArrayList traversals for search, removal, and filtering.

Key Vocabulary
  • ArrayList — Resizable list from java.util; stores reference types (use Integer not int)
  • add(E) — Appends element to end of ArrayList
  • remove(int i) — Removes element at index i and shifts remaining elements left
  • size() — Method (with parens) returning current number of elements in ArrayList
  • ConcurrentModificationException — Runtime error when modifying an ArrayList during enhanced for traversal
Objectives
  • Use ArrayList methods: add, remove, get, set, size.
  • Traverse an ArrayList with indexed and enhanced for loops.
  • Explain when to use ArrayList over an array.
How this can be tested
  • FRQ: write a method that removes elements from an ArrayList based on a condition.
  • MCQ: trace ArrayList operations to find the final contents.
  • MCQ: identify the ConcurrentModificationException scenario.
Practice before moving on
  • Write a method removeNegatives(ArrayList list) that removes all negative values.
  • Trace add and remove operations on a small ArrayList.

2D Arrays

Learn on this site
2D Array Declaration

int[][] grid = new int[rows][cols]; Elements: grid[r][c]. First index is row (0 to rows-1), second is column (0 to cols-1). Each row is itself a 1D array: grid[r] is an int[].

Common misconception: 2D arrays use row-major order: grid[row][col]. Swapping row and column causes logic errors and potential AIOOBE.

How it can be tested: Declare and access 2D arrays; identify row and column dimensions; trace element access.

Traversing 2D Arrays

Nested loop: outer iterates rows, inner iterates columns. grid.length gives number of rows; grid[0].length gives number of columns. Row-major traversal processes each row fully before moving to next row.

Common misconception: Using grid.length for columns instead of grid[0].length causes out-of-bounds for non-square arrays.

How it can be tested: Write nested loops to traverse all elements; compute row sums and column sums.

2D Array Algorithms

Common tasks: sum all elements, find max, find row/column sums, transpose, search. AP® FRQ frequently asks to traverse a 2D array to compute a result.

Common misconception: When searching a 2D array, you need to check all elements — nested loops cannot break out of both at once. Use a boolean flag or return from method when found.

How it can be tested: Write methods that search, sum, or transform 2D array contents.

Key Vocabulary
  • 2D array — Array of arrays; grid[row][col]; first index is row, second is column
  • grid.length — Number of rows in a 2D array
  • grid[0].length — Number of columns (length of first row) in a 2D array
  • row-major order — Traversal that processes all columns of row 0, then all of row 1, etc.
  • jagged array — 2D array where rows have different lengths; not tested heavily on AP® CS A
Objectives
  • Declare and access 2D arrays using row and column indices.
  • Write nested loops for row-major traversal of a 2D array.
  • Compute aggregate values (total sum, row totals) from a 2D array.
How this can be tested
  • FRQ: write a method that processes each element of a 2D array.
  • MCQ: trace 2D array access to find an element or sum.
  • MCQ: identify grid.length vs grid[0].length for rows vs columns.
Practice before moving on
  • Write a method that returns the sum of all elements in a 2D int array.
  • Write a method that checks whether a 2D array is symmetric (grid[r][c] == grid[c][r]).

Array Algorithms and FRQ Practice

Learn on this site
Sequential Search

Traverse from index 0; return index when found; return -1 if not found. Best case O(1); worst case O(n). For unsorted data only.

Common misconception: Return the index, not the value — AP® FRQ often asks for the index of the found element.

How it can be tested: Write a sequential search method; trace which index is returned for a given target.

Selection Sort

Find the minimum in the unsorted portion and swap it to the front. Outer loop selects position; inner loop finds minimum. O(n²) always. Not required to memorize on AP®, but must recognize and trace.

Common misconception: Selection sort is in-place and does not need extra storage. Insertion sort (similar complexity) may also appear in AP® materials.

How it can be tested: Trace selection sort for a small array; identify the number of comparisons and swaps.

FRQ Strategy for Arrays and ArrayLists

Read the problem once for context, once to extract the required algorithm. Write clean loops with correct boundaries. Methods must return the right type. Comment intent when stuck.

Common misconception: AP® graders award partial credit. Writing a correct loop that almost works earns more points than leaving the question blank. Always write something.

How it can be tested: Write FRQ-style methods for array aggregation, filtering, and search.

Key Vocabulary
  • sequential search — Linear scan returning index of target or -1 if not found
  • selection sort — Repeatedly select minimum of unsorted portion and swap to front; O(n²)
  • sentinel value — Special value (-1, null, etc.) indicating absence or end of data
  • partial credit — AP® FRQ awards points for partially correct solutions — always write something
  • in-place — Algorithm that modifies the original array without creating a new one
Objectives
  • Write a sequential search method returning the index of a target.
  • Trace one pass of selection sort on a small array.
  • Write an FRQ-style method that processes array data to return a result.
How this can be tested
  • FRQ: write a complete method that searches an array and returns the index.
  • FRQ: write a method that creates and returns a filtered copy of an ArrayList.
  • MCQ: trace selection sort to find the array state after k passes.
Practice before moving on
  • Write sequentialSearch(int[] arr, int target) returning index or -1.
  • Trace selection sort on {5,2,8,1,4} for the first two passes.
