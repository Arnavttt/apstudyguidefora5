Binary and Data Representation

Learn on this site
Binary Number System

Computers use binary (base 2): only 0 and 1. A bit is one binary digit. 8 bits = 1 byte. Binary to decimal: each bit position is a power of 2. Example: 1011 = 8+0+2+1 = 11.

Common misconception: Binary uses place values (powers of 2), not place values of 10. Memorize powers: 1,2,4,8,16,32,64,128.

How it can be tested: Convert between binary and decimal; calculate the decimal value of a binary string.

Text and Image Representation

Text: each character assigned a binary code (ASCII, Unicode). Unicode supports all world scripts. Images: pixels, each pixel a color code (RGB). More bits per pixel = more possible colors.

Common misconception: A larger pixel count (resolution) and more bits per pixel (color depth) both increase file size. They are independent dimensions of image quality.

How it can be tested: Explain how text and images are stored in binary; calculate file size for a given resolution and color depth.

Limits of Representation

Fixed-size binary representations have limits: integer overflow when result exceeds bit capacity; floating-point rounding (not all decimals can be represented exactly). These are real computing concerns.

Common misconception: Computers cannot represent ALL real numbers exactly. 0.1 in binary is a repeating fraction — financial software must account for this.

How it can be tested: Identify integer overflow and floating-point rounding as limitations of binary representation.

Key Vocabulary
  • bit — Single binary digit: 0 or 1; the smallest unit of data
  • byte — 8 bits; can represent 256 distinct values (2⁸)
  • ASCII — Standard encoding of 128 characters (English letters, digits, symbols) in 7 bits
  • Unicode — International character encoding supporting all world scripts; common encoding: UTF-8
  • overflow — Error when a computed value exceeds the maximum representable by fixed-bit integer
Objectives
  • Convert between binary and decimal.
  • Explain how text, images, and numbers are stored in binary.
  • Identify overflow and rounding as limitations of binary representation.
How this can be tested
  • MCQ: convert a binary value to decimal or identify the binary representation.
  • MCQ: calculate file size for an image with given resolution and color depth.
  • Written response: explain a real-world consequence of floating-point rounding.
Practice before moving on
  • Practice binary↔decimal conversion for 8-bit values.
  • Calculate image file size for various resolution and bit-depth combinations.

Data Compression

Learn on this site
Why Compress?

Compression reduces file size for storage and transmission. Trade-off: processing time vs storage/bandwidth. Lossless preserves all data; lossy permanently discards some data for greater compression.

Common misconception: All compression involves trade-offs. Lossless is reversible; lossy is not — you cannot recover discarded data after lossy compression.

How it can be tested: Explain why data is compressed; identify scenarios requiring lossless vs lossy compression.

Lossless Compression

Lossless algorithms encode patterns to reduce size with no data loss. Run-length encoding (RLE): "AAABBBCC" → "3A3B2C". ZIP files use lossless compression. Text must use lossless.

Common misconception: Lossless compression is not necessarily slower than lossy — it depends on the algorithm and data patterns.

How it can be tested: Apply run-length encoding to a simple sequence; identify lossless file formats.

Lossy Compression

Lossy permanently removes data deemed less important. JPEG for images discards high-frequency detail. MP3 for audio discards frequencies humans hear less well. Smaller size, irreversible.

Common misconception: Applying lossy compression repeatedly degrades quality each time — "generation loss." Save originals in lossless format for editing.

How it can be tested: Explain what lossy compression discards; identify lossy formats; explain generation loss.

Key Vocabulary
  • lossless compression — Reduces file size while preserving all original data; fully reversible
  • lossy compression — Reduces file size by permanently discarding some data; not reversible
  • run-length encoding (RLE) — Lossless method replacing repeated values with count+value: AAA→3A
  • JPEG — Lossy image format; discards fine detail; suitable for photos but not line art
  • PNG — Lossless image format; preserves all pixel data; suitable for graphics and screenshots
Objectives
  • Explain the trade-off between lossless and lossy compression.
  • Apply run-length encoding to a simple data sequence.
  • Choose the correct compression type for a given use case.
How this can be tested
  • MCQ: identify whether a format is lossy or lossless.
  • MCQ: apply RLE to a string and identify the encoded result.
  • Written response: explain why a medical image must use lossless compression.
Practice before moving on
  • Encode and decode a string using run-length encoding.
  • Explain why repeatedly saving a JPEG degrades image quality.

Analog vs Digital and Metadata

Learn on this site
Analog vs Digital

Analog data is continuous (sound waves, temperature). Digital data is discrete — sampled at intervals. Sampling rate: how often analog is measured. Higher sampling rate = better approximation of the original, but larger file.

Common misconception: Digital is not "perfect" — it is an approximation of the analog original. The quality of the approximation depends on sample rate and bit depth.

How it can be tested: Explain sampling and how digital approximates analog; identify trade-offs in sample rate.

Metadata

Metadata is data about data. Photo metadata: timestamp, GPS location, camera settings. File metadata: creator, size, date modified. Metadata enables organization and search but raises privacy concerns.

Common misconception: Metadata can reveal sensitive information even when the main content is private. Photos shared online often contain GPS coordinates in EXIF data.

How it can be tested: Identify types of metadata; explain privacy implications of metadata in shared files.

Data Collection and Privacy

Many apps collect data including metadata. Aggregating data from multiple sources can reveal private information. Users often do not read privacy policies. Legal frameworks (GDPR, COPPA) protect some data.

Common misconception: Even "anonymized" data can be re-identified by combining it with other public data — true anonymization is difficult.

How it can be tested: Explain how data collection raises privacy concerns; identify trade-offs in data sharing.

Key Vocabulary
  • analog — Continuous signal with infinite resolution; e.g., sound waves, temperature
  • digital — Discrete representation of analog data; sampled at fixed intervals
  • sampling rate — How many samples per second of analog signal are captured; higher = better quality
  • metadata — Data describing other data (file size, creation date, GPS coordinates, author)
  • EXIF data — Metadata embedded in photos including camera settings, time, and often GPS location
Objectives
  • Explain how digital systems approximate analog data through sampling.
  • Identify metadata examples and their privacy implications.
  • Describe trade-offs between sample rate and file size.
How this can be tested
  • MCQ: identify the trade-off in choosing a low vs high sampling rate.
  • Written response: explain a privacy risk from metadata.
  • MCQ: distinguish data from metadata in a given scenario.
Practice before moving on
  • Find the EXIF metadata in a phone photo using a metadata viewer.
  • Explain why sharing a photo might reveal your location.

Big Data and Data Analysis

Learn on this site
Big Data Characteristics

Big data: very large data sets requiring special tools. Characteristics: Volume (huge amount), Velocity (fast generation), Variety (structured and unstructured). Traditional databases insufficient.

Common misconception: Big data is not just a large spreadsheet — the scale requires distributed computing (Hadoop, Spark) and different analytical techniques.

How it can be tested: Identify big data characteristics; explain why traditional tools are insufficient for big data.

Data Analysis and Visualization

Data analysis: clean → transform → analyze → visualize. Visualization (charts, graphs, maps) helps identify patterns. Correlation ≠ causation — always question whether an association is causal.

Common misconception: Confusing correlation and causation is one of the most common data analysis errors. Just because two things go up together does not mean one causes the other.

How it can be tested: Distinguish correlation from causation; interpret a graph and identify potential misleading visualizations.

Limitations and Bias in Data

Datasets can be biased if they do not represent the population well. Biased training data produces biased algorithms. Missing data and outliers affect analysis. Data should be questioned critically.

Common misconception: An AI trained on biased data reproduces that bias at scale, often with more authority. Bias in data is a serious ethical issue in computing.

How it can be tested: Identify potential sources of bias in a data set; explain how bias affects algorithmic outputs.

Key Vocabulary
  • big data — Datasets too large for traditional tools; characterized by volume, velocity, variety
  • correlation — Statistical association between two variables; not the same as causation
  • causation — One thing directly causes another; requires more evidence than correlation
  • bias (data) — Systematic skew in a dataset that does not represent the true population
  • data cleaning — Process of fixing errors, handling missing values, and removing outliers before analysis
Objectives
  • Describe characteristics of big data and why special tools are needed.
  • Distinguish correlation from causation with examples.
  • Identify potential sources of bias in a dataset.
How this can be tested
  • Written response: describe how bias in a training dataset affects an AI system.
  • MCQ: distinguish correlation from causation in a described study.
  • MCQ: identify a characteristic of big data.
Practice before moving on
  • Find an example of a misleading correlation from "Spurious Correlations" website.
  • Describe a real-world data bias and its consequences.
