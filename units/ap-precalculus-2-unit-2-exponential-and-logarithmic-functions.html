Exponential Functions

Learn on this site
Exponential Growth and Decay

f(x) = ab^x: base b > 1 → growth; 0 < b < 1 → decay. The base determines the constant multiplicative factor per unit increase in x. Continuous: f(x) = ae^(rx); r > 0 growth, r < 0 decay.

Common misconception: Exponential growth is NOT linear — doubling is a multiplicative relationship, not additive. 2ˣ grows much faster than x².

How it can be tested: Identify growth vs decay from the base; calculate values and interpret percent change.

Properties of Exponential Functions

Domain: all reals; Range: (0,∞) for a>0. y-intercept: (0, a). No x-intercepts (always positive). Horizontal asymptote y=0. Transformations apply as usual (shifts, stretches).

Common misconception: The horizontal asymptote of a basic exponential is y=0, not an x-intercept. The graph gets very close but never reaches zero.

How it can be tested: State domain, range, asymptote, and intercepts; apply transformations to exponential functions.

Compound and Continuous Interest

Compound: A = P(1+r/n)^(nt). Continuous: A = Pe^(rt). More frequent compounding → higher effective rate. Effective annual rate (EAR) = (1+r/n)^n − 1.

Common misconception: Continuous compounding does not give infinite money — it gives a slightly higher amount than annual compounding, approaching the limit e^r per dollar per year.

How it can be tested: Apply compound and continuous interest formulas; compare effective annual rates.

Key Vocabulary
  • exponential function — f(x) = ab^x; base b > 1 → growth; 0 < b < 1 → decay; constant % change per unit x
  • base e — Euler's number ≈ 2.718; base of natural exponential; continuous growth model
  • horizontal asymptote — y = 0 for basic exponential; graph approaches but never reaches 0
  • compound interest — A = P(1+r/n)^(nt); interest computed n times per year
  • continuous growth — A = Pe^(rt); limiting case of compound interest as n→∞
Objectives
  • Identify exponential growth vs decay and state their properties.
  • Apply compound and continuous interest formulas.
  • Describe transformations of exponential functions.
How this can be tested
  • FRQ: model exponential growth/decay from a table or description.
  • MCQ: identify the correct exponential model for a described situation.
  • FRQ: compare compound vs continuous interest.
Practice before moving on
  • Graph f(x) = 2ˣ and g(x) = (1/2)ˣ on the same axes and compare.
  • Compare daily vs continuous compounding for a 5-year investment.

Logarithmic Functions

Learn on this site
Logarithm Definition

log_b(x) = y means b^y = x. Natural log: ln(x) = log_e(x). Common log: log(x) = log_10(x). Inverse of b^x. Key values: log_b(1) = 0; log_b(b) = 1; log_b(b^x) = x; b^(log_b x) = x.

Common misconception: The argument of a logarithm must be positive — log(negative) is undefined. This restricts the domain of log functions.

How it can be tested: Convert between exponential and logarithmic forms; evaluate logs at key values.

Properties of Logarithms

Product: log(AB) = log A + log B. Quotient: log(A/B) = log A − log B. Power: log(A^p) = p·log A. Change of base: log_b(x) = log(x)/log(b). These properties are used to expand or condense log expressions.

Common misconception: These properties require the same base — you cannot mix log_2 and log_5 without converting. Also log(A+B) ≠ log A + log B.

How it can be tested: Expand and condense logarithmic expressions using product, quotient, and power rules.

Graphs of Logarithmic Functions

y = log_b(x): domain (0,∞); range (−∞,∞); x-intercept (1,0); VA x=0. Transformations shift the domain. The log function is the reflection of b^x over y=x.

Common misconception: log_b(x) has a vertical asymptote at x=0, not at y=0. The graph approaches x=0 but extends infinitely in y.

How it can be tested: State domain, range, intercepts, and asymptote; sketch y = log_b(x) and transformations.

Key Vocabulary
  • logarithm — log_b(x) = y means b^y = x; exponent needed to get x from base b
  • natural logarithm ln — log_e(x); inverse of e^x; ln(e^x) = x; e^(ln x) = x
  • common logarithm log — log_10(x); often written log without base; log(10^x) = x
  • change of base formula — log_b(x) = log(x)/log(b) or ln(x)/ln(b); converts any base to a calculable form
  • domain of log — (0, ∞); argument must be strictly positive
Objectives
  • Convert between exponential and logarithmic forms.
  • Apply product, quotient, and power rules of logarithms.
  • State the domain, range, intercepts, and asymptote of logarithmic functions.
How this can be tested
  • FRQ: expand or condense a complex logarithmic expression.
  • MCQ: evaluate a logarithm or identify a domain restriction.
  • FRQ: apply change of base to evaluate log_5(30) on a calculator.
Practice before moving on
  • Expand log₂(8x³/y²) fully using log properties.
  • Evaluate log₃(81) and log₂(1/8) without a calculator.

Solving Exponential and Logarithmic Equations

Learn on this site
Solving Exponential Equations

Strategy: make bases equal (if possible) or take logarithm of both sides. ln(b^x) = x·ln(b). Solve for x by dividing by ln(b). Check for extraneous solutions.

Common misconception: Taking log of both sides is always valid when both sides are positive. If one side could be negative (due to transformation), check the domain first.

How it can be tested: Solve exponential equations by equating bases or applying logarithms; verify solutions.

Solving Logarithmic Equations

Isolate the log, then convert to exponential form. If multiple logs, condense using properties first. Always check: argument of log must be positive after solving.

Common misconception: Extraneous solutions arise when the derived x value makes a log argument non-positive. Always substitute back to verify.

How it can be tested: Solve logarithmic equations; identify and reject extraneous solutions.

Exponential and Log Models

Exponential models: population P(t) = P₀e^(rt), half-life: t₁/₂ = ln(2)/k, doubling time: t_d = ln(2)/r. Solve for unknowns by taking ln. Log models: sound level (decibels), pH = −log[H⁺].

Common misconception: Half-life and doubling time are different values from the rate constant k — compute them separately using ln(2)/k.

How it can be tested: Set up and solve exponential/log models; find doubling time, half-life, or rate from data.

Key Vocabulary
  • one-to-one property — b^a = b^c implies a = c; log_b(a) = log_b(c) implies a = c; used to solve equations
  • extraneous solution — A solution derived algebraically that does not satisfy the original equation
  • doubling time — Time for a quantity to double: t_d = ln(2)/r
  • half-life — Time for a quantity to halve: t_{1/2} = ln(2)/k
  • pH — pH = −log₁₀[H⁺]; logarithmic scale for acidity; decreases by 1 for every 10× increase in [H⁺]
Objectives
  • Solve exponential equations using logarithms.
  • Solve logarithmic equations and check for extraneous solutions.
  • Apply exponential/log models to real-world problems.
How this can be tested
  • FRQ: solve for unknown in an exponential growth model.
  • MCQ: identify an extraneous solution in a log equation.
  • FRQ: find doubling time or half-life from a growth/decay model.
Practice before moving on
  • Solve 3^(x+1) = 50 using natural log; check with a calculator.
  • Find the half-life of a substance decaying at 8% per year.

Regression and Modeling with Functions

Learn on this site
Choosing a Model

Select the best function type by analyzing the data. Linear: constant first differences. Quadratic: constant second differences. Exponential: constant ratios. Logarithmic: rapid then slowing growth. Evaluate fit with r² (coefficient of determination).

Common misconception: A high r² does not guarantee the model is correct — it only measures linear correlation. Always check a residual plot and whether the model makes contextual sense.

How it can be tested: Determine the appropriate regression model from a data table; interpret r² and residuals.

Exponential Regression

Use data to find y = ab^x. Linearize by taking log: log(y) = log(a) + x·log(b). On a calculator: ExpReg or enter log(y) values for linear regression then convert. Extrapolation beyond data range is unreliable.

Common misconception: Extrapolating an exponential model far beyond the data range produces unreliable predictions — exponential growth does not continue indefinitely in the real world.

How it can be tested: Perform or interpret exponential regression; extrapolate with appropriate caution.

Piecewise and Composition of Functions

Piecewise functions use different rules for different domain pieces. Function composition: (f∘g)(x) = f(g(x)). Order matters: f∘g ≠ g∘f in general. Domain of f∘g: valid inputs for g that produce outputs in domain of f.

Common misconception: Composition order: f∘g means apply g first, then f. Many students reverse the order.

How it can be tested: Evaluate and graph piecewise functions; compute and interpret function compositions.

Key Vocabulary
  • residuals — Differences between observed data and predicted model values; should show no pattern
  • r² (coefficient of determination) — Measures proportion of variance explained by model; 1 = perfect fit; 0 = no fit
  • exponential regression — Finding ab^x that best fits data; linearized by taking log
  • piecewise function — Function defined by different rules on different intervals of the domain
  • function composition — (f∘g)(x) = f(g(x)): apply g first, then apply f to the result
Objectives
  • Choose the appropriate function type to model a dataset.
  • Interpret r² and residual plots for model quality.
  • Evaluate function compositions and piecewise functions.
How this can be tested
  • FRQ: given a table of values, determine the best model type and justify.
  • MCQ: identify whether a dataset is better fit by linear or exponential regression.
  • FRQ: compute f(g(x)) for given functions.
Practice before moving on
  • Use a graphing calculator to find exponential regression for a dataset.
  • Evaluate and graph a piecewise function with three pieces.
